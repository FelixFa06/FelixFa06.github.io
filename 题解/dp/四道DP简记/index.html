11.1模拟 四道DP简记

搬运自 11 月洛谷博客。

hb 5月~6月 的时候给我找了刘汝佳紫皮书上 30 道远古 dp 题，原题都是 UVA 的。当时饱受这套题折磨。

csp 前 hb 选了其中 4 道比较有价值的题，让我出成模拟。但是我太弱了什么都不会，所以在这里简单记录一下这套题的思路。

T1 UVA 12170.Easy Climb

传送门

Description

有 nnn 座山，编号从 111 到 nnn 。每座山有高度 hih_ihi​ 。一次操作中，可以选定一座山 2≤i≤n−12\leq i \leq n-12≤i≤n−1 ，令 hi←hi−1h_i\gets h_i-1hi​←hi​−1 或 hi←hi+1h_i\gets h_i+1hi​←hi​+1 。同时，操作规定不能把山的高度修改为负数。（操作不能改变第一座山和最后一座山的高度）。

要求操作结束后任意相邻两座山的高度差绝对值不大于 ddd 。即 ∀1≤i<n,∣hi−hi+1∣≤d\forall 1\leq i<n,|h_i-h_{i+1}|\leq d∀1≤i<n,∣hi​−hi+1​∣≤d 。求达到这一条件的最小操作次数。如果无解，输出 −1-1−1 。

2≤n≤300,0≤d,hi≤1092\le n \le 300,0\le d,h_i\le 10^92≤n≤300,0≤d,hi​≤109

Solution

不难想到一个朴素的 dp 状态：

  • dpi,jdp_{i,j}dpi,j​ 表示确定了前 iii 座山，第 iii 座修改后高度为 jjj 的最小代价。

转移方程：

dpi,j=min⁡k=max⁡(0,j−d)j+d{dpi−1,k}+∣j−hi∣dp_{i,j}=\min_{k=\max(0,j-d)}^{j+d}\{dp_{i-1,k}\}+|j-h_i| dpi,j​=k=max(0,j−d)minj+d​{dpi−1,k​}+∣j−hi​∣

设 ddd 和 hih_ihi​ 的值域大小为 HHH 。转移复杂度 O⁡(H)\operatorname{O}(H)O(H) ，总复杂度 O⁡(nH2)\operatorname{O}(nH^2)O(nH2) 。

由于转移状态时，是取上一行一段连续区间的最值，且这个区间的左右端点都随 jjj 递增，所以可以用单调队列优化。总复杂度 O⁡(nH)\operatorname{O}(nH)O(nH)。

现在的问题就是，值域 HHH 很大，不能直接作为 dp 的状态。所以我们就会思考，这么大的值域当中，是否有一些值是最优策略下一定不会用到的呢？换句话说，设修改后的高度为 hi′h_i'hi′​，是否存在一个元素较少集合 SSS 使得最优策略中 ∀hi′,hi∈S\forall h_i',h_i\in S∀hi′​,hi​∈S。如果存在这样的 SSS ,我们就可以修改第二维状态是否时间复杂度能过关。

经过一系列猜结论，我们或许可以发现最后策略中：

∀hi,hi′∈S={hj+x×d∣1≤j≤n,−n≤x≤n}\forall h_i,h_i'\in S=\{h_j+x\times d|1\le j\le n,-n\le x\le n\} ∀hi​,hi′​∈S={hj​+x×d∣1≤j≤n,−n≤x≤n}

这个结论不难用归纳法证明：

考虑修改后最小的 hi′h_i'hi′​ ，它已经是最小了，说明在修改时它向下减小一定不优，那么它一定是向上增加到某个临界高度后停止。而这个临界高度就是 min⁡(hi−1′,hi+1′)−d\min(h_{i-1}',h_{i+1}')-dmin(hi−1′​,hi+1′​)−d。那么我们分开考虑 ≤i−1\le i-1≤i−1 和 ≥i+1\ge i+1≥i+1 的部分，继续选取其中最小的 hj′h_j'hj′​ 进行类似归纳，到最后只剩下第一座和最后一座山，这两座山不能修改。对于所有 hih_ihi​ ，若它不需要修改，则 hi′=hih_i'=h_ihi′​=hi​，否则 hi′=min(hi−1′,hi+1′)−dh_i'=min(h_{i-1}',h_{i+1}')-dhi′​=min(hi−1′​,hi+1′​)−d，而 hi−1h_{i-1}hi−1​ 和 hi+1h_{i+1}hi+1​ 一定也属于上述集合 SSS 。所以 ∀hi,hi′∈S={hj+x×d∣1≤j≤n,−n≤x≤n}\forall h_i,h_i'\in S=\{h_j+x\times d|1\le j\le n,-n\le x\le n\}∀hi​,hi′​∈S={hj​+x×d∣1≤j≤n,−n≤x≤n}。

因为 ∣S∣=O⁡(n2)|S|=\operatorname{O}(n^2)∣S∣=O(n2)，所以我们用 SSS 中元素排序离散化后的编号作为 dp 状态，就使复杂度从 O⁡(nH)\operatorname{O}(nH)O(nH) 变为 O⁡(n3)\operatorname{O}(n^3)O(n3)。

Code

  • 时间复杂度：O⁡(n3)\operatorname O(n^3)O(n3)
  • 空间复杂度：O⁡(n2)\operatorname O(n^2)O(n2)

Summary

本题重点和思维难点在于对 dp 状态的优化方法，用提炼关键点集合的思路解决。

Tips

本题还有 O⁡(nlog⁡n)\operatorname O(n\log n)O(nlogn) 的 凸包优化 dp 做法。

T2 UVA1439.Exclusive Access 2

传送门

Description

这题原题的题意复杂程度实在让人难绷，所以我把抽象化后的题意写一下：

一张无向图（不保证连通）。要对每条边确定一个方向。使其成为有向图。

要求定向后形成的有向图中，最长路径的长度尽可能小。你输出最长路径长度的 最小值-1，并输出每条边的方向。（图上一条路径的长度该为路径所经过的边的数量）

1≤n≤15,0≤m≤100,1≤xi,yi≤n1\le n\le 15,0\le m\le 100,1\leq x_i, y_i\leq n1≤n≤15,0≤m≤100,1≤xi​,yi​≤n。

Solution

这题乍一看像个图论题，不太好 dp。既然正着不太好考虑，我们就尝试倒推对边定向的过程。我们假设现在已经得到了定向后的有向图。设所有边定向后，对每个点 iii，以 iii 结尾的最长路长度为 lenilen_ileni​。

然后根据 lenilen_ileni​ 的值对所有点分层，即 lenilen_ileni​ 相同的点在同一层。反证法可以证明，同一层中的节点之间不存在边。

也就是说如果我们先对图中所有点赋 lenlenlen 值并分层，若满足每一层内部任意两点之间不存在边，那么这组 lenlenlen 值是合法的，且一定存在对应定向方案。而最长路长度就是 max⁡{leni}\max\{len_i\}max{leni​}，即层数-1。

这样问题就从边定向转化成为将节点合法地分层。

而点数 nnn 很小。明摆着让我们状压dp。

  • dpmaskdp_{mask}dpmask​ 表示点集为 maskmaskmask，最长路的最小值。

转移时，每次枚举 maskmaskmask 的一个子集 sss 为图中最后一层，要求 sss 中任意两点之间没有边。

dpmask=min⁡{dpmask∖s}+1dp_{mask}=\min\{dp_{mask\setminus s}\}+1 dpmask​=min{dpmask∖s​}+1

在 dp 前对每个集合预处理是否两两之间没有边。

最后答案为 dp{1,2,...,n}−2dp_{\{1,2,...,n\}}-2dp{1,2,...,n}​−2。 dp 的同时记录路径，最后顺着路径标出每个点的层号，根据层号大小对所有边定向。

code

  • 时间复杂度：O⁡(n22n+3n)\operatorname O(n^22^n+3^n)O(n22n+3n)
  • 空间复杂度：O⁡(2n)\operatorname O(2^n)O(2n)

Summary

本题比较考研思维，难点是通过对定向后的图中节点分层，来转换成状压 dp 的问题。

T3 UVA1204.Fun Game

传送门

Description

有若干个字符（至少 2 个）围成一个圈，每个字符可能是 ‘B’ 或 ’G‘。

现在已知 nnn 个由 ‘B’ 和 ‘G’ 构成的字符串。它们都是通过在圈上选一个位置，顺时针或逆时针遍历若干个字符得到的。求圈中字符数的最小值。

1≤n≤16,1≤∣si∣≤1001\le n\le 16,1\le|s_i|\le 1001≤n≤16,1≤∣si​∣≤100

Solution

首先，我们发现若一个字符串是另一个字符串（正反任意）的子串，那么删去这个字符串不会对条件有影响。删去这样的字符串。

保留剩下字符串后，目标的环形字符串一定是由这些字符串按照一定的正反和排列顺序，分别取一些部分拼接而成。

将每个字符串正反各记录一边，考虑 O(n2∣s∣)O(n^2|s|)O(n2∣s∣) 预处理每两个字符串 sis_isi​和 sjs_jsj​ 中，最长的 lenlenlen，使得 sis_isi​ 的长度为 lenlenlen 的后缀 =sj=s_j=sj​长度为 lenlenlen 的前缀，即最大首尾重合部分。这样拼接时能最小化长度，用 KMP 或者哈希能在线性时间内完成。

nnn 范围很小，又是明摆着让我们状压 dp。

  • dpmask,i,0/1dp_{mask,i,0/1}dpmask,i,0/1​ 表示当前已存在子串集合为 maskmaskmask，上一个放的字符串的是 s[i]s[i]s[i]（000 表示正，111 表示反）。

每次枚举下一个摆的字符串编号及其正反，根据预处理的重合长度计算代价并转移，记录路径。

最后对所有 dp{1...n},i,0/1dp_{\{1...n\},i,0/1}dp{1...n},i,0/1​，根据路径倒推出对应字符串。再对其自身求一遍 KMP，找出最长的 lenlenlen，使得长度为 lenlenlen 的前缀 === 长度为 lenlenlen 的后缀，将首位相接，长度减少 lenlenlen，就得到合法的最短环形字符串。

取所有环形字符串长度最小值 ansansans，答案为 max(2,ans)max(2,ans)max(2,ans)。

Code

  • 时间复杂度：O⁡(n2∣si∣+n22n)\operatorname O(n^2|s_i|+n^22^n)O(n2∣si​∣+n22n)
  • 空间复杂度：O⁡(n2+∑∣si∣+n2n)\operatorname O(n^2+\sum|s_i|+n2^n)O(n2+∑∣si​∣+n2n)

Summary

我感觉这题的思维难度不及前两题，但是这题在维护字符串的各种操作技巧上要求比较高，代码难度也较高，稍有不慎就会挂掉。

T4 UVA1375. The Best Name for Your Baby

传送门

Description

你有一个字符串，一开始是为 ”S 。现在定义 nnn 个规则，每个规则规定一个大写字母可以转换成一个字符串，例如 'A’ →\to→ “abCd”。现在要求通过这 nnn 个规则去变换字符串，使得其中全部是小写字母。求能得到的长度为 lll 的字符串中字典序最小的。将其输出。

1≤n≤50,0≤l,∣si∣≤201\le n\le 50,0\le l,|s_i|\le 201≤n≤50,0≤l,∣si​∣≤20

Solution

注意到这题的数据范围非常小，我们可以尝试直接用 dp 数组存字典序最小的字符串。根据题目定义，无穷大的字符串就规定为 “-”。现在要考虑如何用 dp 规划字符串之间的转换关系。

  • 设 gc,jg_{c,j}gc,j​ 表示大写字符 ccc 能转换得到的长度为 jjj 的字典序最小字符串。

但是这样一个状态不能诠释 nnn 条规则的转换关系。不妨再设一个状态。

  • 设 fid,i,jf_{id,i,j}fid,i,j​ 表示在第 ididid 条规则对应的字符串中，考虑长度为 iii 的前缀，能转换得到的长度为 jjj 的字典序最小字符串。

这样一来就可以 dp 了。转移式很容易想到：

gc,j=min⁡{fid,∣sid∣,j}fid,i,j={fid,i−1,j−1+cfid,i−1,j−k+gc,kg_{c,j}=\min\{f_{id,|s_{id}|,j}\}\\ f_{id,i,j}=\begin{cases}f_{id,i-1,j-1}+c\\f_{id,i-1,j-k}+g_{c,k}\end{cases} gc,j​=min{fid,∣sid​∣,j​}fid,i,j​={fid,i−1,j−1​+cfid,i−1,j−k​+gc,k​​

dp 的复杂度为 O⁡((n+∑∣si∣)×l2)\operatorname O((n+\sum|s_i|)\times l^2)O((n+∑∣si​∣)×l2) 。而一次字符串操作的复杂度为 O⁡(l)\operatorname O(l)O(l)，所以总复杂度 O⁡((n+∑∣si∣)×l3)\operatorname O((n+\sum|s_i|)\times l^3)O((n+∑∣si​∣)×l3)。

但是仔细观察转移方程，发现事情并没有这么简单。转移的过程中是会出现环的，例如了下面两条规则：

{S→AA→S\begin{cases}S\to A\\A\to S\end{cases} {S→AA→S​

如果这样的情况存在，那么 g′A′,1g_{'A',1}g′A′,1​ 和 g′S′,1g_{'S',1}g′S′,1​ 之间就会相互转移，形成了环。这样 dp 就不能直接粗暴地转移。

但是我们思考后发现，这种情况只会出现在 jjj 相同的情况下，不同的大写字母之间。也就是说，如果我们按照 jjj 把 dp 值堪称若干层，那么环只会出现在同一层间。对于每一层 jjj，可以先从 j−1j-1j−1 层转移过来，然后再处理层内部的转移。

我们在此之前预处理 cango[x][y]cango[x][y]cango[x][y] 表示大写字母 xxx 能否转化为大写字母 yyy 。如果可以，那么 gx,j=gy,jg_{x,j}=g_{y,j}gx,j​=gy,j​，处理出这个之后，将 gc,jg_{c,j}gc,j​ 视为节点，若 xxx 可以转换到 yyy，则连一条边权为 000 的无向边。我们就可以对同一层所有节点用类似 Bellman-Ford 最短路的方式进行若干次松弛，就能处理环上的转移。

这样的做法时间复杂度不变，仍然是 O⁡((n+∑∣si∣)×l3)\operatorname O((n+\sum|s_i|)\times l^3)O((n+∑∣si​∣)×l3)。

Code

  • 时间复杂度：O⁡((n+∑∣si∣)×l3)\operatorname O((n+\sum|s_i|)\times l^3)O((n+∑∣si​∣)×l3)
  • 空间复杂度：O⁡((n+∑∣si∣)×l2)\operatorname O((n+\sum|s_i|)\times l^2)O((n+∑∣si​∣)×l2)

Summary

这道题目和 T3 一样是和字符串有关的 dp 。思维难度上同样要求不高，但是对代码能力的要求又高出一个量级。在写代码的时候，只要对 dp 的过程和处理字符串转换关系的细节稍有不清楚，就很容易把程序写错。在开始写代码前可以整理自己需要的变量、数组、函数，理出大致的分工和步骤再开始写。
